# 产品说明书

## 一、产品定位
Motila 是一套面向内部管理场景的后台系统，基于 NestJS + React + Ant Design 构建，提供用户、权限、模块、系统配置等基础能力，并支持按配置快速扩展。

## 二、适用人群
- 技术管理者（快速搭建管理后台）
- 研发团队（基于现有骨架扩展业务模块）
- 运维/实施人员（进行配置化维护）

## 三、核心功能总览

### 1）认证与账号
- 用户注册 / 登录
- JWT 鉴权
- 获取当前登录用户信息（`/auth/me`）
- 修改当前用户密码
- 忘记密码（邮箱找回，`/auth/forgot-password`）：支持用户名或邮箱提交，**统一返回泛化提示，不暴露账号是否存在**
- 图形验证码接口：`GET /auth/captcha?scene=login|register|forgotPassword`
- 认证流程接入验证码：风控命中后返回 `needCaptcha`，前端自动展示验证码输入区并校验通过后再提交

### 1.1）认证风控（新增）
- 新增“配置 -> 风控配置”菜单，集中管理登录 / 注册 / 忘记密码三类场景的风控策略
- 配置项支持：总开关、Redis 不可用降级策略、分场景阈值、黑白名单、版本发布与回滚
- 登录 / 注册 / 忘记密码接口已接入运行期风控入口，支持按 IP / 账号做频率限制与失败次数封禁
- 支持按 IP 或按账号重置运行期风控计数与封禁（可选场景）
- 运行依赖 Redis；当 Redis 不可用时，可按配置选择“允许请求（降级）”或“阻断请求”
- 降级策略为 `ALLOW_WITH_CAPTCHA` 时，开发环境会回退到内存计数，避免风控功能完全失效

### 2）用户管理
- 用户列表、创建、编辑、删除
- 用户角色分配
- 编辑用户支持用户名唯一性校验，编辑回显完整
- 编辑布局：主字段 4 列展示，头像相关字段 2 列展示
- 用户头像：
  - 字段：`avatarImage`（上传/base64）、`avatarUrl`
  - 支持上传与预览
  - 编辑页头像预览采用圆形显示风格
  - 展示优先级：`avatarImage` > `avatarUrl`

### 2.1）个人邮箱配置（新增）
- 支持 QQ / 163 邮箱配置
- 可维护：邮箱账号、授权码、SMTP/IMAP 主机与端口、SSL
- 支持“测试配置”按钮（SMTP 连接 + 认证 + 测试邮件）
- 授权码后端加密存储，接口返回脱敏显示

### 3）权限与角色（RBAC）
- 角色管理、权限绑定
- 用户与角色关系维护
- 权限点驱动菜单与页面访问
- 分配用户场景支持全量拉取（`pageSize=0`）

### 4）模块管理
- 模块启用/禁用
- 模块状态影响菜单可见性和路由访问

### 5）审计日志
- 记录关键管理动作（如用户、角色、配置变更）
- 支持审计日志查询

### 6）系统配置（配置 -> 系统配置）
支持字段：
- `name`：系统名称
- `title`：系统标题
- `logoUrl`：Logo 地址
- `logoImage`：Logo 图片（base64）
- `footerText`：页脚文案（支持部分 Markdown 链接）
- `isActive`：是否生效

业务规则：
- 同一时刻仅允许一条配置处于生效状态（`isActive=true`）

前端联动：
- 左侧 Logo
- 浏览器标签标题（`document.title`）
- Footer 页脚文案
- 登录页浏览器标题：优先读取生效系统配置的 `title`，无配置时默认 `Welcome to Motila`
- 首屏默认标题：`Welcome to Motila`（避免加载阶段显示 `frontend`）

Logo 展示优先级：
1. `logoImage`
2. `logoUrl`
3. 默认文案 `Motila`

编辑体验：
- 编辑/新建采用页面内标准编辑界面（非弹窗）
- 支持删除已上传 Logo 图片（清空 `logoImage`）
- 支持手动清空 `logoImage` 文本后保存删除
- 兼容创建/编辑接口差异：`removeLogoImage` 仅在编辑(PATCH)提交，避免创建(POST)校验报错
- 后端请求体大小限制：`2mb`（用于 base64 Logo 上传）

### 7）主题与界面
### 7.0）未登录首页（新增）
- 系统未登录状态默认展示品牌首页
- 首页采用进一步极简化的终端风格首屏：去除顶部品牌标题、登录按钮与“立即开始”按钮
- `Hello World` 上方新增终端式动态打字机输入效果：`WELCOME TO MOTILA.`
- 首页仅保留一个“进入系统”主按钮；顶部 `博文 / 文档 / 社区` 仍为预留导航
- 登录与注册已拆分为独立路由页面：`/login`、`/register`
- 登录/注册切换时会同步更新浏览器路由，支持直接访问与刷新保留当前认证页

- 支持 business / tech / dark / auto 四种皮肤
- 顶部用户区域显示为「头像 + 昵称 + 角色」
- 后台左侧菜单顶部不再显示纯文字系统名；若未配置 Logo，则保持空白极简样式
- 移动端导航采用 Drawer 抽屉菜单：
  - 小屏隐藏左侧菜单
  - 顶部菜单按钮唤起抽屉
  - 菜单点击后自动关闭抽屉

### 7.1）博客管理（新增）
- 博客模块拆分为「博客分类 / 博客文章」两个独立菜单与列表页面
- 创建/编辑统一为独立页面（与人员管理一致），不再使用弹窗
- 列表点击行进入详情页，详情页可进入编辑页
- 博客文章列表支持勾选多条记录后批量发布，批量发布后文章状态切换为“已发布”并生成发布时间

### 8）菜单配置化
菜单项支持：
- `parentKey`：父子级绑定
- `sortOrder`：排序

系统通过配置构建菜单树并渲染，减少硬编码层级逻辑。

### 9）域名访问控制（Vite）
- 前端开发服务启用 Host 校验
- 已允许域名：
  - `yangtiancheng.cn`
  - `www.yangtiancheng.cn`
- 用途：避免通过域名访问时触发 Vite 拒绝请求

## 四、权限模型补充
与本轮功能相关的新增权限点：
- `settings.read`：查看系统配置
- `settings.update`：修改系统配置
- `risk.read`：查看风控配置
- `risk.update`：修改风控配置

未具备权限时，菜单隐藏且相关路由不可访问。

## 五、非功能特性
- 安全：
  - 未登录访问受保护接口返回 401
  - 管理员保护策略（防止误删/误降权）
- 可维护：前后端分层清晰，文档同步更新
- 可扩展：菜单、模块、权限均支持配置化演进

### 10）项目管理与人员管理（独立页面流转）
- 新建与查看不再放在列表页下方内嵌区域，改为独立页面：
  - 项目：`/projects/create`、`/projects/:id`、`/projects/:id/edit`
  - 人员：`/hr/employees/create`、`/hr/employees/:id`、`/hr/employees/:id/edit`
- 列表行点击进入详情页，详情页进入编辑页，避免列表页过长。

### 11）列表批量删除交互统一
- 用户/项目/人员列表批量按钮统一命名为“删除”。
- 勾选记录后点击删除，弹框确认：`要删除当前选中的N条记录吗？`。
- 仅在点击“确认”后执行删除，点击“取消”不删除。

### 12）刷新留页能力优化
- 修复项目管理/人员管理页面 F5 刷新后偶发被误判为无权限并跳转仪表盘的问题。
- 优化后：在权限与模块状态加载完成前不做拦截，刷新后可保持在当前业务页。

### 13）权限配置（新版）
- 列表页提供：多选、创建、删除（系统内置角色不可删除）
- 点击角色行进入角色详情/编辑页，编辑页包含“保存/返回”按钮
- 权限点支持跨模块混搭勾选，不再出现切换模块后已选项被清空
