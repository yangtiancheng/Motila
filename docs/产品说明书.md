# 产品说明书

## 一、产品定位
Motila 是一套面向内部管理场景的后台系统，基于 NestJS + React + Ant Design 构建，提供用户、权限、模块、系统配置等基础能力，并支持按配置快速扩展。

## 二、适用人群
- 技术管理者（快速搭建管理后台）
- 研发团队（基于现有骨架扩展业务模块）
- 运维/实施人员（进行配置化维护）

## 三、核心功能总览

### 1）认证与账号
- 用户注册 / 登录
- JWT 鉴权
- 获取当前登录用户信息（`/auth/me`）
- 修改当前用户密码

### 2）用户管理
- 用户列表、创建、编辑、删除
- 用户角色分配
- 用户头像：
  - 字段：`avatarImage`（上传/base64）、`avatarUrl`
  - 支持上传与预览
  - 展示优先级：`avatarImage` > `avatarUrl`

### 3）权限与角色（RBAC）
- 角色管理、权限绑定
- 用户与角色关系维护
- 权限点驱动菜单与页面访问
- 分配用户场景支持全量拉取（`pageSize=0`）

### 4）模块管理
- 模块启用/禁用
- 模块状态影响菜单可见性和路由访问

### 5）审计日志
- 记录关键管理动作（如用户、角色、配置变更）
- 支持审计日志查询

### 6）系统配置（配置 -> 系统配置）
支持字段：
- `name`：系统名称
- `title`：系统标题
- `logoUrl`：Logo 地址
- `logoImage`：Logo 图片（base64）
- `footerText`：页脚文案（支持部分 Markdown 链接）
- `isActive`：是否生效

业务规则：
- 同一时刻仅允许一条配置处于生效状态（`isActive=true`）

前端联动：
- 左侧 Logo
- 浏览器标签标题（`document.title`）
- Footer 页脚文案

Logo 展示优先级：
1. `logoImage`
2. `logoUrl`
3. 默认文案 `Motila`

说明：系统配置编辑场景支持 Logo 上传并持久化回显。

### 7）主题与界面
- 支持 business / tech / dark / auto 四种皮肤
- 顶部用户区域显示为「头像 + 昵称 + 角色」

### 8）菜单配置化
菜单项支持：
- `parentKey`：父子级绑定
- `sortOrder`：排序

系统通过配置构建菜单树并渲染，减少硬编码层级逻辑。

## 四、权限模型补充
与本轮功能相关的新增权限点：
- `settings.read`：查看系统配置
- `settings.update`：修改系统配置

未具备权限时，菜单隐藏且相关路由不可访问。

## 五、非功能特性
- 安全：
  - 未登录访问受保护接口返回 401
  - 管理员保护策略（防止误删/误降权）
- 可维护：前后端分层清晰，文档同步更新
- 可扩展：菜单、模块、权限均支持配置化演进
