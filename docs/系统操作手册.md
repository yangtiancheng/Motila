# 系统操作手册

## 一、启动与访问

### 1. 本地开发启动
```bash
cd Motila
npm install
npm --prefix backend install
npm --prefix frontend install
npm run dev:all
```

### 2. 访问地址
- 前端：`http://<主机IP>:5173`
- 后端健康检查：`http://<主机IP>:3000/health`

### 3. 健康检查标准
- 以后端 `GET /health` 返回 `{"ok":true}` 作为服务可用标准
- `GET /` 返回 404 不代表服务异常

---

## 二、登录与账号
1. 使用用户名 + 密码登录
2. 如忘记密码：点击登录页「忘记密码」，填写用户名（可选邮箱二次校验），系统会发送临时密码到用户已绑定邮箱
3. 登录后可在右上角进入：
   - 个人中心
   - 修改密码
   - 退出登录

---

## 三、用户管理

### 1. 进入路径
`配置 -> 用户管理`

### 2. 新建用户
1. 点击「新建用户」
2. 填写：用户名、邮箱、昵称、头像上传/地址（可选）、密码、角色
3. 点击保存

### 3. 编辑用户
1. 在列表点击「编辑」
2. 修改信息后保存

### 4. 头像说明
- 字段：`avatarImage`（上传/base64）、`avatarUrl`
- 支持上传与预览
- 展示优先级：`avatarImage` > `avatarUrl`
- 无头像时系统显示昵称首字母占位

### 5. 常见限制
- 不能删除自己
- 不能把自己降为普通用户
- 不能删除/降权最后一个管理员

---

## 四、系统配置管理

### 0. 个人邮箱配置（个人信息）
- 路径：右上角 -> 个人信息
- 支持 QQ / 163 邮箱配置
- 填写：邮箱账号、授权码/密码、SMTP/IMAP 主机端口、SSL
- “测试配置”会验证 SMTP 连接 + 认证，并发送测试邮件
- 授权码后端加密存储，页面只显示脱敏内容

### 1. 进入路径
`配置 -> 系统配置`

### 2. 字段说明
- 系统名称（name）
- 系统标题（title）
- Logo 地址（logoUrl，可选）
- Logo 上传（logoImage）
- Footer 文案（footerText，可选，支持 `[文本](链接)`）
- 是否生效（isActive）

### 3. 生效机制
- 同时只允许一个配置生效
- 某条配置设为生效时，其他配置自动失效

### 4. 前端联动点
- 左侧 Logo
- 浏览器标题
- Footer 文案
- 登录页浏览器标题：优先使用生效系统配置 `title`，无配置时默认 `Welcome to Motila`
- 首屏默认标题：`Welcome to Motila`（避免加载阶段显示 `frontend`）

### 5. Logo 优先级
1. `logoImage`
2. `logoUrl`
3. 默认文案 `Motila`

### 6. 编辑与删除 Logo 图片
- 编辑/新建在页面内编辑区完成（非弹窗）
- 点击“删除已上传Logo”可清空 `logoImage`
- 也支持手动清空 `logoImage` 文本域后保存（后端将显式删除图片）
- 接口差异说明：`removeLogoImage` 仅在编辑(PATCH)提交；新建(POST)不提交该字段，避免校验报错
- 保存后立即生效（若当前配置为启用状态）

### 7. 上传大小限制说明
- 后端 JSON / URL-Encoded 请求体限制为 `2mb`
- `logoImage` 为 base64 时体积会膨胀，建议上传压缩后图片

### 1. 权限控制
- 菜单可见性与页面访问由权限决定
- 系统配置相关权限：
  - `settings.read`
  - `settings.update`

### 2. 模块开关
- 模块禁用后，对应菜单和页面不可访问

## 六、菜单配置说明
菜单配置文件支持：
- `parentKey`：定义父菜单
- `sortOrder`：定义同级排序

调整菜单层级和顺序时，优先修改配置项，避免改渲染逻辑。

### 移动端导航使用说明
- 手机访问时，左侧菜单改为自动隐藏
- 顶部左侧出现菜单按钮（☰）
- 点击按钮打开抽屉菜单，选择功能页后抽屉自动关闭
- 若菜单未更新，先执行浏览器强刷（Ctrl+F5 或清缓存）

## 七、长期运行（PM2）

### 1. 启动
```bash
pm2 start ecosystem.config.cjs
pm2 status
pm2 save
```

### 2. 重启
```bash
pm2 restart motila-backend motila-frontend
pm2 save
```

> 若当前环境无 pm2（报 `pm2: command not found`），可先用命令行直接启动前后端进程。

## 八、域名访问（Vite Host）

当通过域名访问前端出现以下报错时：

```text
Blocked request. This host ("yangtiancheng.cn") is not allowed.
```

处理步骤：
1. 编辑 `frontend/vite.config.ts`
2. 在 `server.allowedHosts` 中加入域名：
   - `yangtiancheng.cn`
   - `www.yangtiancheng.cn`
3. 本地开发场景补充 `server.proxy`：将 `/api/*` 转发到 `http://127.0.0.1:3000/*`
4. 重启前端服务（或 PM2 对应进程）
5. 刷新页面验证

示例配置：
```ts
export default defineConfig({
  plugins: [react()],
  server: {
    allowedHosts: ['yangtiancheng.cn', 'www.yangtiancheng.cn'],
    proxy: {
      '/api': {
        target: 'http://127.0.0.1:3000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },
})
```

## 九、问题排查

1. **分配用户弹窗报错 `pageSize must not be less than 1`**
   - 已修复后端分页参数校验，支持 `pageSize=0` 的全量查询
   - 若仍报错，请确认服务已重启并加载最新代码

2. **前端能打开但接口报错**
   - 先检查后端 `GET /health`
   - 再检查前端请求地址是否为 `http://<当前主机>:3000`

3. **系统配置保存后未生效**
   - 检查该条配置是否 `isActive=true`
   - 页面刷新后再次确认

4. **头像不显示**
   - 优先检查 `avatarImage` 是否为空
   - 若走 `avatarUrl`，检查是否为合法 URL 且可访问

5. **Footer 链接未生效**
   - 检查格式是否为 `[文本](https://...)`
