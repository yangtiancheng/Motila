# 系统操作手册

## 一、启动与访问

### 1. 本地开发启动
```bash
cd Motila
npm install
npm --prefix backend install
npm --prefix frontend install
npm run dev:all
```

### 2. 访问地址
- 前端：`http://<主机IP>:5173`
- 后端健康检查：`http://<主机IP>:3000/health`

### 3. 健康检查标准
- 以后端 `GET /health` 返回 `{"ok":true}` 作为服务可用标准
- `GET /` 返回 404 不代表服务异常

---

## 二、登录与账号
1. 使用用户名 + 密码登录
2. 登录后可在右上角进入：
   - 个人中心
   - 修改密码
   - 退出登录

---

## 三、用户管理

### 1. 进入路径
`配置 -> 用户管理`

### 2. 新建用户
1. 点击「新建用户」
2. 填写：用户名、邮箱、昵称、头像上传/地址（可选）、密码、角色
3. 点击保存

### 3. 编辑用户
1. 在列表点击「编辑」
2. 修改信息后保存

### 4. 头像说明
- 字段：`avatarImage`（上传/base64）、`avatarUrl`
- 支持上传与预览
- 展示优先级：`avatarImage` > `avatarUrl`
- 无头像时系统显示昵称首字母占位

### 5. 常见限制
- 不能删除自己
- 不能把自己降为普通用户
- 不能删除/降权最后一个管理员

---

## 四、系统配置管理

### 1. 进入路径
`配置 -> 系统配置`

### 2. 字段说明
- 系统名称（name）
- 系统标题（title）
- Logo 地址（logoUrl，可选）
- Logo 上传（logoImage）
- Footer 文案（footerText，可选，支持 `[文本](链接)`）
- 是否生效（isActive）

### 3. 生效机制
- 同时只允许一个配置生效
- 某条配置设为生效时，其他配置自动失效

### 4. 前端联动点
- 左侧 Logo
- 浏览器标题
- Footer 文案

### 5. Logo 优先级
1. `logoImage`
2. `logoUrl`
3. 默认文案 `Motila`

---

## 五、权限与模块

### 1. 权限控制
- 菜单可见性与页面访问由权限决定
- 系统配置相关权限：
  - `settings.read`
  - `settings.update`

### 2. 模块开关
- 模块禁用后，对应菜单和页面不可访问

---

## 六、菜单配置说明
菜单配置文件支持：
- `parentKey`：定义父菜单
- `sortOrder`：定义同级排序

调整菜单层级和顺序时，优先修改配置项，避免改渲染逻辑。

---

## 七、长期运行（PM2）

### 1. 启动
```bash
pm2 start ecosystem.config.cjs
pm2 status
pm2 save
```

### 2. 重启
```bash
pm2 restart motila-backend motila-frontend
pm2 save
```

> 若当前环境无 pm2（报 `pm2: command not found`），可先用命令行直接启动前后端进程。

## 八、域名访问（Vite Host）

当通过域名访问前端出现以下报错时：

```text
Blocked request. This host ("yangtiancheng.cn") is not allowed.
```

处理步骤：
1. 编辑 `frontend/vite.config.ts`
2. 在 `server.allowedHosts` 中加入域名：
   - `yangtiancheng.cn`
   - `www.yangtiancheng.cn`
3. 重启前端服务（或 PM2 对应进程）
4. 刷新页面验证

示例配置：
```ts
export default defineConfig({
  plugins: [react()],
  server: {
    allowedHosts: ['yangtiancheng.cn', 'www.yangtiancheng.cn'],
  },
})
```

## 九、问题排查

1. **分配用户弹窗报错 `pageSize must not be less than 1`**
   - 已修复后端分页参数校验，支持 `pageSize=0` 的全量查询
   - 若仍报错，请确认服务已重启并加载最新代码

2. **前端能打开但接口报错**
   - 先检查后端 `GET /health`
   - 再检查前端请求地址是否为 `http://<当前主机>:3000`

3. **系统配置保存后未生效**
   - 检查该条配置是否 `isActive=true`
   - 页面刷新后再次确认

4. **头像不显示**
   - 优先检查 `avatarImage` 是否为空
   - 若走 `avatarUrl`，检查是否为合法 URL 且可访问

5. **Footer 链接未生效**
   - 检查格式是否为 `[文本](https://...)`
