# 系统操作手册

## 一、启动与访问

### 1. 本地开发启动
```bash
cd Motila
npm install
npm --prefix backend install
npm --prefix frontend install
npm run dev:all
```

### 2. 访问地址
- 前端：`http://<主机IP>:5173`
- 后端健康检查：`http://<主机IP>:3000/health`

### 3. 健康检查标准
- 以后端 `GET /health` 返回 `{"ok":true}` 作为服务可用标准
- `GET /` 返回 404 不代表服务异常

## 二、登录与账号
## 一点五、未登录首页
1. 用户未登录直接访问系统时，会先看到系统首页
2. 首页采用极简终端风格首屏，点击右上角「立即开始」进入登录页
3. 顶部「博文 / 文档 / 社区」当前为预留入口，点击后仅提示暂未开放
4. 登录页路径为 `/login`，注册页路径为 `/register`
5. 在登录页与注册页之间切换时，浏览器地址会同步变化，刷新后仍停留当前认证页
6. 若需要找回密码，仍可通过登录页中的“忘记密码”完成

1. 使用用户名 + 密码登录
2. 如忘记密码：点击登录页「忘记密码」，填写用户名或邮箱（二选一即可），系统会在账号信息存在且已绑定邮箱时发送临时密码邮件（对外不暴露是否存在）
3. 若风控命中，需要先完成图形验证码验证（点击验证码图片可刷新）
4. 登录后可在右上角进入：
   - 个人中心（可修改昵称、用户名、头像地址）
   - 修改密码
   - 退出登录

## 三、用户管理

### 1. 进入路径
`配置 -> 用户管理`

### 2. 新建用户
1. 点击「新建用户」
2. 填写：用户名、邮箱、昵称、头像上传/地址（可选）、密码、角色
3. 点击保存

### 3. 编辑用户
1. 在列表点击需要修改的用户行进入编辑页
2. 修改信息后保存
3. 规则：用户名必须唯一；表单主字段按 4 列展示，头像相关字段按 2 列展示
4. 手机端如列表列较多，可左右滑动表格查看完整内容
5. 创建/编辑类页面顶部应固定提供保存/取消等操作按钮；若缺失，优先检查 Card 头部样式是否被隐藏

### 4. 未登录首页与公开博客
1. 未登录访问 `/` 时，首页仅展示极简 Hello World 品牌入口
2. 点击顶部「博文」按钮，跳转到独立公开博客页 `/blog`
3. 公开博客页支持按博客分类筛选文章
4. 支持关键词搜索与分页切换
5. 点击文章卡片可在博客页下方打开沉浸式详情阅读区域
6. 页面底部 Footer 文案读取系统配置中的 Footer 文字

### 4. 头像说明
- 字段：`avatarImage`（上传/base64）、`avatarUrl`
- 支持上传与预览
- 编辑页头像预览为圆形显示风格
- 展示优先级：`avatarImage` > `avatarUrl`
- 无头像时系统显示昵称首字母占位

### 5. 常见限制
- 不能删除自己
- 不能把自己降为普通用户
- 不能删除/降权最后一个管理员

## 四、系统配置管理

### 0. 个人邮箱配置（个人信息）
- 路径：右上角 -> 个人信息
- 支持 QQ / 163 邮箱配置
- 填写：邮箱账号、授权码/密码、SMTP/IMAP 主机端口、SSL
- “测试配置”会验证 SMTP 连接 + 认证，并发送测试邮件
- 授权码后端加密存储，页面只显示脱敏内容

### 1. 进入路径
`配置 -> 系统配置`

### 2. 字段说明
- 系统名称（name）
- 系统标题（title）
- Logo 地址（logoUrl，可选）
- Logo 上传（logoImage）
- Footer 文案（footerText，可选，支持 `[文本](链接)`）
- 是否生效（isActive）

### 3. 生效机制
- 同时只允许一个配置生效
- 某条配置设为生效时，其他配置自动失效

### 4. 前端联动点
- 左侧 Logo
- 浏览器标题
- Footer 文案
- 登录页浏览器标题：优先使用生效系统配置 `title`，无配置时默认 `Welcome to Motila`
- 首屏默认标题：`Welcome to Motila`（避免加载阶段显示 `frontend`）

### 5. Logo 优先级
1. `logoImage`
2. `logoUrl`
3. 默认文案 `Motila`

### 6. 编辑与删除 Logo 图片
- 编辑/新建在页面内编辑区完成（非弹窗）
- 点击“删除已上传Logo”可清空 `logoImage`
- 也支持手动清空 `logoImage` 文本域后保存（后端将显式删除图片）
- 接口差异说明：`removeLogoImage` 仅在编辑(PATCH)提交；新建(POST)不提交该字段，避免校验报错
- 保存后立即生效（若当前配置为启用状态）

### 7. 上传大小限制说明
- 后端 JSON / URL-Encoded 请求体限制为 `2mb`
- `logoImage` 为 base64 时体积会膨胀，建议上传压缩后图片

### 1. 权限控制
- 菜单可见性与页面访问由权限决定
- 系统配置相关权限：
  - `settings.read`
  - `settings.update`

### 2. 模块开关
- 模块禁用后，对应菜单和页面不可访问

### 3. 登录 / 注册 / 忘记密码风控
- 路径：`配置 -> 风控配置`
- 支持配置场景：登录、注册、忘记密码
- 每个场景可维护：
  - 启用开关
  - 每分钟 / 每小时 / 每天阈值
  - 失败多少次触发验证码
  - 失败多少次封禁
  - 封禁时长
  - Retry-After 秒数
- 支持维护：IP 白名单、账号白名单、IP 黑名单、账号黑名单
- 支持操作：保存草稿、发布、回滚版本
- 支持按 IP 或账号重置运行期风控状态（可选场景）
- 运行说明：已接入 auth 接口风控入口；Redis 可用时执行频率限制与失败封禁；Redis 不可用时按降级策略执行
- 降级策略为 `ALLOW_WITH_CAPTCHA` 时，开发环境回退内存计数并触发验证码校验

## 六、菜单配置说明
菜单配置文件支持：
- `parentKey`：定义父菜单
- `sortOrder`：定义同级排序

调整菜单层级和顺序时，优先修改配置项，避免改渲染逻辑。

### 移动端导航使用说明
- 手机访问时，左侧菜单改为自动隐藏
- 顶部左侧出现菜单按钮（☰）
- 点击按钮打开抽屉菜单，选择功能页后抽屉自动关闭
- 若菜单未更新，先执行浏览器强刷（Ctrl+F5 或清缓存）

## 八、域名访问（Vite Host）

当通过域名访问前端出现以下报错时：

```text
Blocked request. This host ("yangtiancheng.cn") is not allowed.
```

处理步骤：
1. 编辑 `frontend/vite.config.ts`
2. 在 `server.allowedHosts` 中加入域名：
   - `yangtiancheng.cn`
   - `www.yangtiancheng.cn`
3. 本地开发场景补充 `server.proxy`：将 `/api/*` 转发到 `http://127.0.0.1:3000/*`
4. 重启前端服务（或 PM2 对应进程）
5. 刷新页面验证

示例配置：
```ts
export default defineConfig({
  plugins: [react()],
  server: {
    allowedHosts: ['yangtiancheng.cn', 'www.yangtiancheng.cn'],
    proxy: {
      '/api': {
        target: 'http://127.0.0.1:3000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },
})
```

## 九、问题排查

1. **分配用户弹窗报错 `pageSize must not be less than 1`**
   - 已修复后端分页参数校验，支持 `pageSize=0` 的全量查询
   - 若仍报错，请确认服务已重启并加载最新代码

2. **前端能打开但接口报错**
   - 先检查后端 `GET /health`
   - 再检查前端请求地址是否为 `http://<当前主机>:3000`

3. **系统配置保存后未生效**
   - 检查该条配置是否 `isActive=true`
   - 页面刷新后再次确认

4. **头像不显示**
   - 优先检查 `avatarImage` 是否为空
   - 若走 `avatarUrl`，检查是否为合法 URL 且可访问

5. **Footer 链接未生效**
   - 检查格式是否为 `[文本](https://...)`

6. **头像维护交互**
   - 点击头像区域可直接上传/替换头像
   - 头像右上角叉号可一键清除头像
   - 上传后支持即时预览

7. **项目/人员管理新交互**
   - 新建/编辑采用页面内标准编辑卡片，不再使用弹窗
   - 列表点击任意数据行可直接进入编辑
   - 支持多选后批量删除（复制/失效按钮暂占位）
8. **用户管理新交互**
   - 用户列表点击行可直接进入编辑
   - 列表支持多选后批量删除
   - 编辑页头像位于表单最上方
9. **发件中心**
   - 在左侧菜单进入“发件中心”
   - 输入收件人、主题、正文后发送
   - 下方可查看发送日志与失败原因
10. **收件箱同步**
   - 在发件中心点击“同步邮件”触发 IMAP 拉取
   - 收件箱列表支持点击行查看邮件详情

## 八、项目管理与人员管理（新版页面流）

### 1. 项目管理
- 列表入口：`项目管理`
- 新建项目：点击“新建项目”进入 `/projects/create`
- 查看项目：在列表点击任意行进入 `/projects/:id`
- 编辑项目：在项目详情点击“编辑”进入 `/projects/:id/edit`

### 2. 人员管理
- 列表入口：`人员管理`
- 新建员工：点击“新建员工”进入 `/hr/employees/create`
- 查看员工：在列表点击任意行进入 `/hr/employees/:id`
- 编辑员工：在员工详情点击“编辑”进入 `/hr/employees/:id/edit`

### 3. 批量删除（统一交互）
- 在用户/项目/人员列表勾选多条记录后，点击“删除”
- 系统弹框确认：`要删除当前选中的N条记录吗？`
- 点击“确认”执行删除；点击“取消”不删除

### 4. 刷新后页面跳转说明
- 现已修复：在项目管理/人员管理页面按 F5，不应再误提示“无权限或模块关闭”并跳转仪表盘
- 若仍出现，请检查：
  1) 当前账号是否具备对应读权限（`project.read` / `hr.read`）
  2) 对应模块是否启用（`project` / `hr`）

## 九、权限配置（新版）

### 1. 列表页
- 路径：`配置 -> 权限配置`
- 可执行：创建、勾选多选、删除（系统内置角色禁删）
- 点击行可进入角色编辑页

### 2. 编辑页
- 路径：`/settings/rbac/:code`（创建为 `/settings/rbac/create`）
- 页面右上提供“保存”“返回”按钮
- 权限点可跨模块混搭选择，切换模块不会丢失已选权限

11. **风控配置发布后未生效**
   - 确认已点击“发布”，而不只是“保存草稿”
   - 确认当前账号具备 `risk.read` / `risk.update` 权限
   - 确认模块 `risk-control` 已启用
   - 若配置依赖 Redis，请确认后端已正确配置 `REDIS_URL` 或 `REDIS_HOST`
